# DjangoForEveryone

<!-- by Liang Zegui -->
A Django tutorial project repository for beginners, containing basic application development examples and phased tutorial code to help users systematically learn the Django framework.

✨ Project Features  
🏗️ Built with Django 4.0+  
📚 Branch-managed tutorial progress (e.g., post-tutorial branch contains complete code)  
🧩 Modular design with core example app "baseapp"  
🖥️ MySQL database configuration examples  
📊 Integrated template system and static resource management  
📝 Detailed development documentation and sample code  
🚀 Suitable for beginners and intermediate developers  

🚀 Quick Start  
Clone repository & switch branch  
```bash
git clone https://github.com/yourname/DjangoForEveryone.git
cd DjangoForEveryone
git checkout post-tutorial  # Switch to completed branch

Environment Setup
Ensure Python 3.10+ is installed and create virtual environment:

bash
pip install -r requirements.txt
Start Server

bash
python manage.py runserver
Visit http://127.0.0.1:8000/ to view the project.

## Annotate the Project Structure and Its Corresponding Functions

Files and Folders in the Root Directory
DjangoForEveryone/
├── README.md # The project's documentation file, usually containing the project introduction, usage methods, architecture, etc. Here it introduces that this project is used for JimShapedCoding's Django tutorial, as well as the basic architecture of the project and the way to view other contents.
├── baseapp/ # A Django application, used to explain and display relevant topics in the tutorial.
├── venv/ # The Python virtual environment directory, used to isolate the project's dependencies and avoid dependency conflicts between different projects.

Directory of the baseapp Application
baseapp/
├── .idea/ # The configuration folder automatically generated by IDEs such as IntelliJ IDEA or PyCharm, containing IDE-specific settings for the project.
├── baseapp/ # This is the configuration directory of the Django project, containing the project's setting files.
│ ├──settings.py/ # The main setting file of the project, including database configuration, application list, static file paths, etc.
│ ├──urls.py/ # The URL routing configuration file of the project, defining the URL mapping relationships of the project.
│ ├──wsgi.py/ # The WSGI server configuration file for deploying the project
├── main/ # This is a Django application directory, which may contain view, model, template and other files of this application.
│ ├──models.py/ # Define the data models of the application, that is, the database table structure.
│ ├──views.py/ # View functions or classes that handle user requests and return responses.
│ ├──urls.py/ # The URL routing configuration file of the application, defining the URL mapping relationships within the application.
├── static/ # Store the static files of the project, such as CSS, JavaScript, images, etc. These files are directly referenced in the project and do not require processing by the server.
├── templates/ # Store the HTML template files of the project, used to generate dynamic web pages. Django combines data and templates through the template engine to generate the final HTML pages.
├── db.sqlite3/ # The SQLite database file used by default in the Django project, used to store the project's data.
├── manage.py/ # The management script of the Django project, used to execute various management commands, such as creating applications, running the development server, performing database migrations, etc.

Directory of the venv Virtual Environment
venv/
├── Lib/ # The directory of Python library files installed in the virtual environment.
├── Scripts/ # The script directory of the virtual environment, containing scripts for activating and deactivating the virtual environment, as well as shortcuts to the Python interpreter.
├── pyvenv.cfg # The configuration file of the virtual environment, containing basic information and configuration options of the virtual environment.

Directory for Placing Images
images/

🎯 Project Goals and Use Cases
The goal of this project is to provide an easy-to-use learning resource for Django beginners while serving as a reference for professional project structures for intermediate developers.

Use Cases:

Learning the basics and advanced techniques of the Django framework.
Developing small web applications or rapid prototypes.
Serving as an example project in teaching and training environments.
